# 剑指Offer

这是一本主要服务于求知面试的经典书籍，书中内容非常丰富，道理、技巧很多，适合应届毕业生复习基础知识，突击面试。
这本书的语音，比较通俗易懂，看着非常轻松舒适，可能和自己有些基础的原因，做着书中面试编程题，感觉还是比较轻松的，但是
收获非常多，自己代码和参考答案一对比，瞬间倍受打击，确实逊色太多了。这本书内容很多，非常全面，主要偏向于实战，理论讲解，
配套面试题，实战演练，我非常喜欢，也是唯一一本，让我老老实实写代码的书，其他计算机语言书籍，我一般粗略看看，
很少实际动手编程，不喜欢思考，看着好像都会，自己完全独立的写、思考，又完全不会。。。

#### 存在的问题很多：
	
	容易忘记错误检查
	非法参数检查
	输入异常检查
	
	考虑常常不周全，容易有逻辑漏洞
	**高效算法，知道少，需要努力提高**
	
#### 总括：这本书有7章，48道面试题，涉及方面非常广，特别是第二章，概括了所有基础知识。

第一章：面试的流程，了解即可

第二章：面试的基础知识，讲解非常全面和丰富，面试题非常精巧，给力！
	
	编程语言
	
	数据结构
	
	算法
	
	三大块就概括了所有基础知识，非常全面，涉及知识非常多，学习这一章，是非常好的知识回忆、复习、巩固过程。

第三章：高质量的代码

	主要包括了规范性、完整性、鲁棒性
	代码不仅要逻辑思路清楚，边界条件、异常输入、指针越界等都同等重要，既要保证主体逻辑框架，还得保证完整性、鲁棒性。
	复杂的链表操作，树遍历都是最基本的指针操作，必须要掌握！

第四章：解决面试题的思路

	学会画图非常重要，既可以帮助自己寻找思路，答案，也能方便向面试官讲解自己的思路
	
	举例子，使得抽象问题具体化，容易分析、解决，找到突破口。
	
	分解：复杂问题，想到极端情况，最简单情况去处理，分而治之策略

第五章：优化时间和空间效率

	时间效率，一般应用而言，时间效率更加重要，内存越来越大，也更加不值钱，时间就是生命。
	
	对于嵌入式系统开发而言，一般存储空间有限，更加注重空间效率。
	
	对于面试问题，得具体问题具体分析，多与面试官沟通，了解清楚题目要求，然后做相应的分析，优化
	
	空间换时间策略：哈希表、记录表法、动态规划等。
	
第六章：面试中的各项能力

	沟通能力：良好的沟通能力，无论面试还是工作都是需要的。
	
	学习能力：对于难题，不要放弃，多提问，请求提示，一步一步解决。不仅解决了面试问题，还能学到很多自己本来不会的知识。
		敢于提问，善于提问，强烈的求知欲望。
	
	举一反三能力：会使用知识迁移，特别是连续的两个问题，是否存在关系，一般面试的第一题，可能只是引导，第二题则是第一题
		的引申，升入。
	
	抽象建模能力：把生活中的模型转化为程序语言表达出来。
	
	发散思维能力：当常规方法不能使用的时候，必须突破思维局限，打破常规，想出新奇的方法。
	要求：基本功扎实、知识面要广，如此才有希望想到好方法。
	
	厚积博发：一点点努力，一点点进步！
	
	求职面试，就是推销自己的过程！
	
第七章：两个面试案例

	字符串转换成整数，看似简单的面试题，一定要沉着、冷静，不能骄傲大意，处理好异常情况、越界情况、等边界条件
	
	树中两个节点的最低公共祖先，对于复杂问题，也要冷静，不要畏惧困难，不懂的就问，多与面试官沟通，请求提示，指导，
	一步一步解决问题。
	
	三大法宝：画图、举例、分解

#### 收获感触

这是一本讲解程序员面试的书，通过50个面试题，形象具体的展现了面试的那些事儿。分析了面试流程，方法，技巧，对于即将毕业的应届生，
正在找工作的面试者，都是非常有用的一本书，非常感谢作者的精彩整理和分享。

全书按照面试的额流程展开，各章节依据实际面试题做引导，实践性非常强，所有面试题，都给出了参考代码，最重要的是，都有解题思路的分析，
讲解，特别分析了各种解题，代码实现技巧，对比了各种实现的优劣，和适用场合。

这是第一本我敲完了所有代码的，C/C++方面书，由于以前有一些微弱的C/C++语言基础，所以我看了好几本这方面的书，都不喜欢敲代码，只喜欢看，
课后习题做的也不多，很多实际没有做，只是看书去了，计划着看第二遍的时候，敲代码。

理想是丰满的，现实是骨感的。看书需要时间，敲代码需要更多的时间，所以还没有看第二遍的书。。。

敲代码确实话费更多的时间，毕竟看书只是扫一眼，就ok了，自然看书看得快，实际忘记的更快！因此对比效率而已，慢慢敲代码。才是王道。

也看了一些大牛的分享学习经验，都是强调看书慢慢看，代码都要自己敲一遍，才能收获更多，实现更多。

敲完了一本书的代码，也值得高兴一下，第一次完成的，不容易啊，总算有了0的突破！

**路漫漫其修远兮，吾将上下而求索**
